# 💾 데이터 저장 개선 가이드

## 🎯 목표:
회원 정보와 비밀번호를 안전하게 저장하고 관리

---

## 📊 현재 상황:

### ❌ 문제점:
```
1. 메모리 DB 사용
   → 서버 재시작 시 모든 데이터 삭제

2. 비밀번호 평문 저장
   → 보안 취약 (해킹 시 그대로 노출)

3. Railway 무료 플랜 제약
   → 파일 저장 불가
```

### ✅ 확인 방법:
```javascript
// backend/database-simple.js
this.users = [];  // 메모리 배열
this.orders = []; // 메모리 배열

// 서버 재시작하면:
this.users = [];  // 다시 빈 배열!
```

## 🔧 해결 방법 1: 로컬 서버 + SQLite (추천!)

### 장점:
- ✅ 모든 데이터 파일에 저장
- ✅ 서버 재시작해도 데이터 유지
- ✅ 비밀번호 암호화 가능
- ✅ 무료!
- ✅ 백업 쉬움 (파일 복사)

### 단점:
- ❌ PC 꺼지면 주문 불가
- ❌ 외부 접속은 Cloudflare Tunnel 필요

### 구현 방법:

#### 1단계: bcrypt 설치
```bash
cd backend
npm install bcrypt
```

#### 2단계: database.js 사용
```javascript
// backend/server.js
const db = require('./database');  // database-simple.js → database.js
```

#### 3단계: 비밀번호 암호화
```javascript
// backend/database.js에 이미 구현됨!
const bcrypt = require('bcrypt');

async createUser(phone, name, email, address, password) {
  const hashedPassword = await bcrypt.hash(password, 10);
  // ... DB에 hashedPassword 저장
}
```

#### 4단계: 로컬 서버 실행
```bash
START.bat

외부 접속:
START-EXTERNAL.bat
```

## 🔒 비밀번호 암호화 중요성:

### 평문 저장 (현재):
```javascript
{
  phone: "010-1234-5678",
  password: "mypassword123"  // ← 그대로 보임!
}

해커가 DB 탈취하면:
→ 모든 비밀번호 그대로 노출!
```

### 암호화 저장 (권장):
```javascript
{
  phone: "010-1234-5678",
  password: "$2b$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy"
}

해커가 DB 탈취해도:
→ 비밀번호 알 수 없음!
→ 복호화 불가능!
```
