### 1. 서버 로그 (가장 중요!)

---

## 📍 Railway에서 로그 확인하는 방법

### 방법 1: Railway 대시보드에서 확인

1. **Railway 대시보드 접속**
   - https://railway.app 접속
   - 로그인

2. **프로젝트 선택**
   - 배포한 프로젝트 클릭

3. **서비스 선택**
   - 웹 서버 서비스 클릭 (Node.js 서비스)

4. **Deployments 탭 클릭**
   - 상단 메뉴에서 "Deployments" 선택

5. **최신 배포 클릭**
   - 가장 위에 있는 최신 배포 클릭

6. **Logs 탭 클릭**
   - 배포 상세 페이지에서 "Logs" 탭 클릭

7. **로그 확인**
   - 스크롤해서 다음 메시지 찾기:

---

## ✅ 확인해야 할 로그 메시지

### PostgreSQL 연결 성공 시:
```
✅ PostgreSQL 데이터베이스 사용 (Railway)
🔄 PostgreSQL 데이터베이스 초기화 중...
✅ PostgreSQL 테이블 생성 완료
✅ 메뉴 초기화 완료
```

### SQLite 사용 시 (PostgreSQL 연결 실패):
```
✅ SQLite 데이터베이스 사용 (로컬)
```

---

## 🔍 로그에서 확인할 내용

### 1. 데이터베이스 연결 확인
- "PostgreSQL 데이터베이스 사용" 메시지가 보이면 성공!
- "SQLite 데이터베이스 사용"이 보이면 PostgreSQL 연결 실패

### 2. 테이블 생성 확인
- "PostgreSQL 테이블 생성 완료" 메시지 확인
- 에러가 없으면 정상

### 3. 서버 시작 확인
- "Server running on port 3000" 같은 메시지 확인
- 서버가 정상적으로 시작되었는지 확인

---

## 🚨 문제가 있을 때

### PostgreSQL 연결 실패 시:
1. Railway → Variables 탭 확인
2. `DATABASE_URL` 환경 변수가 있는지 확인
3. PostgreSQL 서비스가 실행 중인지 확인

### 에러 메시지가 보일 때:
- 에러 내용을 복사해서 확인
- 주로 연결 오류나 테이블 생성 오류

---

## 📸 스크린샷 위치

Railway 대시보드:
```
프로젝트 → 서비스 → Deployments → 최신 배포 → Logs
```
